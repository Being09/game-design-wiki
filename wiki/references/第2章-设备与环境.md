# 第2章 目标设备与开发环境

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

做小游戏的第一道门槛：选设备与开发环境。作者的建议很明确——**无特殊偏好的话，首选 PC 浏览器**。现代浏览器是成熟的游戏平台，发个 URL 别人就能玩，传播成本最低。开发环境的选择更多取决于"你喜欢什么"，而非游戏规模（但做 2D 小游戏用 Unreal Engine 属于杀鸡用牛刀）。

## 浏览器开发选项

| 选项 | 说明 | 代表 |
|---|---|---|
| **JavaScript + 库** | 浏览器原生语言，用库省去渲染等繁琐工作 | p5.js、Phaser |
| **幻想主机** | 轻量一体化引擎（含角色/关卡/声音编辑器），刻意限制功能，适合小项目 | PICO-8、Pyxel、TIC-80 |
| **游戏引擎** | 功能全面，可导出浏览器(HTML5)，也能发 PC/移动 | Unity、GameMaker、Godot |
| **Maker 工具** | 面向特定类型（RPG、视觉小说） | RPG Maker MV、WOLF RPG Editor、TyranoBuilder |
| **可视化编程** | 适合不习惯文本编程者，含预制图文音 | Scratch、MakeCode |
| **其他语言** | 借 WebAssembly 跑非 JS，但编译复杂，建议借幻想主机/引擎过渡（Pyxel→Python，Unity→C#） |

## 关键建议

- **触控兼容**：做浏览器游戏时确保可用触摸（PC 上即鼠标）操作，兼顾移动端，扩大受众。
- **创作动机不必来自"易传播"**：为特定设备创作本身的趣味也是正当动机——做 PC/移动 App、用 SmileBASIC 4 做 Switch 游戏、在掌机上跑 PICO-8 都行。"自由地创作"。

## 工具速查（按推荐场景）

- 新手即开即玩：p5.js（web editor + learn 教程）
- 想要限制感的小项目：PICO-8（教育版 + 手册）
- 要多平台发布：Godot / Unity / GameMaker
- 不写代码：Scratch / MakeCode（MakeCode 还能跑在掌机硬件上）
