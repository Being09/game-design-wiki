# 游戏数学基础

> 综合多本书的跨书主题文章（主要来源：《游戏中的数学与物理学》）。

## 定义

**游戏数学基础**是游戏开发所需的初等数学工具集：数字表达（整数/浮点/二进制）、代数（方程求解、函数图）、几何与三角（角度、三角形、三角函数）、向量与矩阵、微积分（微分、积分、ODE）。它们是运动建模、碰撞检测、变换、寻路等所有游戏算法的数学地基。

## 多本书视角对比

- **《游戏中的数学与物理学》**（[参考](../references/游戏中的数学与物理学（第2版）.md)）：第 1 部分（第 1-6 章）系统讲解。从数字的计算机表达（整数、浮点、舍入误差）出发，经代数运算、几何三角、向量，到矩阵与微积分，为后续物理与算法部分奠定工具基础。强调"累积式"编排——每个数学工具在后续都有应用场景。
- **《代码本色》**（[参考](../references/代码本色.md)）：Shiffman 不系统讲数学基础，而是在用到时即用即讲（向量在第1章、三角在第3章振荡、概率在第0章）。偏"按需引入"。

## 方法/模型

- **数字表达**：整数/浮点的计算机表示，舍入误差与精度问题，二进制/十六进制。
- **代数**：等式配平、二次/三次方程、联立方程、函数图与参数曲线。
- **几何与三角**：角度与弧度、毕达哥拉斯定理、三角函数与恒等式、正弦/余弦定理、相似三角形。
- **向量与矩阵**：见 [[向量与运动]]、[[矩阵与变换]]。
- **微积分**：函数梯度、微分与积分、常微分方程（ODE）、数值近似（划界法、梯度法）。

## 具体例子：浮点陷阱

**浮点精度**是游戏数值 bug 的头号来源。`0.1 + 0.2` 在 IEEE 754 双精度下 ≠ 0.3：

```
0.1 + 0.2 = 0.30000000000000004
```

因为 0.1 在二进制是无限循环小数，被截断。后果：
- **累积误差**：每帧 `position += velocity * 0.1`，跑一万帧后位置漂移几像素。
- **相等判断失败**：`if (hp == 0)` 永远不为真（hp 是 0.0000001），角色死不掉或死两次。
- **解法**：用 `if (abs(a-b) < epsilon)` 判等；货币用整数（分）而非浮点（元）；物理用定点数。

**三角函数的运动应用**：

| 函数 | 游戏用途 |
|---|---|
| sin/cos | 圆周运动（x=r·cos t, y=r·sin t）、振荡（来回摆动）、波浪 |
| atan2(y,x) | 求朝向角（A 到 B 的方向） |
| sin²+cos²=1 | 单位化（向量除以模） |

**圆周运动**（角色绕中心旋转）：`x = cx + r·cos(θ); y = cy + r·sin(θ)`，θ 每帧 += 角速度。比"算切线再移动"简单得多。

**atan2 求朝向**：A 朝 B 瞄准，`angle = atan2(B.y−A.y, B.x−A.x)`。注意用 atan2 而非 atan——atan 分不清象限，atan2(y,x) 用两分量符号自动判象限。

**微积分在游戏里**：
- 微分 = 速度（位置的导数）、加速度（速度的导数）。
- 积分 = 累积伤害、曲线下面积、资源累积。
- ODE（常微分方程）= 物理模拟核心（F=ma 是二阶 ODE，欧拉法数值求解）。

**毕达哥拉斯定理**算距离（最常用）：`distance = sqrt((x2−x1)² + (y2−y1)²)`。但开方很贵，比较距离时常用平方距离（`dist² < r²`）省掉 sqrt——性能敏感代码的标准优化。

## 应用

- 浮点精度影响物理稳定性与数值平衡（见批次②数值公式体系）。
- 三角函数用于旋转、振荡、圆周运动。
- 微分用于速度/加速度、最优化（梯度法）。
- 函数图用于理解成长曲线、伤害曲线（见批次②成长曲线设计）。
