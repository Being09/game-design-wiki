# 第7章 自制游戏开发工具

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

让游戏创作更愉悦的一条路：**自制工具**（游戏库、像素画工具、作曲工具，乃至幻想主机/引擎/编程语言）。工具与游戏开发交替进行能带来新层次的乐趣，但有"沉溺工具、游戏烂尾"的风险。

## 为何自制工具

1. **获得高度熟悉、完全理解内部运作的工具**——自己造的，自然知道怎么用出想要的结果。
2. **自由添加喜爱功能**，API/UI 设计贴合个人偏好，开发环境舒适。
3. **工具创作本身带来乐趣**——与游戏（边做边探索终态）不同，工具目的相对明确，是游戏开发中好的调剂，看功能逐步扩展很有成就感。

## 风险：双刃剑

业界名言："**Don't build a game engine, build a game.**"（别造引擎，造游戏）——警惕沉迷周边工具而荒废游戏本体导致烂尾。**保持平衡**才能让工具创作为游戏开发增色。

## 案例一：crisp-game-lib（浏览器小游戏库）

"How to make your own game engine (and why)" 列出引擎应有功能。作者据此做 [crisp-game-lib](https://github.com/abagames/crisp-game-lib)（TypeScript）：

**设计哲学**：约 **200 行实现一个游戏**。一个游戏只需定义 `update()` 函数。

**内置功能**：
- 选声音名即播音效。
- 设 `isPlayingBgm: true` 自动生成 BGM。
- 设 `isReplayEnabled: true` 启用重放。
- 定义 `theme` 切换外观（复古 CRT、像素画风）。
- `characters` 数组轻松定义像素画。

> 思考"符合自己开发风格的库该有哪些功能"，自造原创库是高效又愉悦的方式。

## 案例二：crisp-game-lib-portable（DIY 掌机移植）

DIY 掌机趋势：以 Arduino 等单板微控制器为核心 + 屏幕 + 按键。优势是按喜好造主机；劣势是设备碎片化、无通用机型。掌机游戏多用 C，故需把库移植到 C。

**[crisp-game-lib-portable](https://github.com/abagames/crisp-game-lib-portable)**：把原 TS 库移植到 C，支持 M5StickC PLUS、Adafruit PyBadge，借 Emscripten 也能跑浏览器。

**跨设备架构**：库核心 + 游戏代码对所有设备（含浏览器）通用，仅需少量设备特定代码（约 250 行）：
- 设备初始化（如 Arduino `setup()`）调 `initGame()`。
- 帧更新（如 `loop()`）调 `setButtonState()` 通知按键状态 + 调 `updateFrame()`。
- 在 `machineDependent.h` 定义绘图与声音。

**移植难点**：PyBadge 仅能用 `analogWrite()` 调蜂鸣器音量，需手动管理发声时机、生成方波；像素画 `md_drawCharacter()` 需配合缓存（hash + LGFX_Sprite 配对）。

## 案例三：PEEKPOKE（自制幻想主机）

**幻想主机**=开发者"要是有这样的游戏机就好玩了"的 whimsical 想法落地。典范 PICO-8（128x128 屏/4 声/256 个 8x8 精灵/Lua 开发环境+编辑器）。造 PICO-8 级别的很难，但"造自己的幻想主机"很有诱惑力——需**刻意限制规格**。

**[PEEKPOKE](https://github.com/abagames/peekpoke)**：极简幻想主机，PC/手机浏览器可玩。

**设计精髓**：**所有主机控制仅靠 peek/poke 命令读写内存**（内存映射 I/O）：
- 32x30 像素 8 色屏、8x5 文字屏、7 键输入、单音蜂鸣器。
- **无内置开发环境**，游戏用 JavaScript 脚本。

> 把主机特性全交给内存 + 开发环境外包给 JS，构建过程大幅简化。能做的游戏较基础，但勉强算幻想主机；加启动 splash screen 增添主机感。

## 核心论点

无开发环境时造幻想主机的工作量大减。"幻想有有趣规格的主机并轻松造出来"本身就很好玩。考虑造致敬现有主机（Cassette Vision、ATARI 2600）或有异色特性的主机也很有趣。
