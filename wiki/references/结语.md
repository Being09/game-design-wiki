# 结语

> 综合自《In Conclusion》（ABA Games, CC BY 4.0）。原文见 [joys-of-small-game-development 仓库](https://github.com/abagames/joys-of-small-game-development-en)。

全书从小游戏的乐趣、创作方法到 AI 自动生成的可能，纵览了小游戏开发的各类话题。结语不做新论证，而是回到起点：**鼓励读者动手做**。

## 核心鼓励

- 创作和游玩小游戏都很容易——幻想主机与游戏引擎丰富了创作环境，新手教程与文档充沛，起步惊人地简单。
- 做出游戏后，**发布上网看反应**很有趣；**参加 Game Jam** 分享作品、收反馈，能进一步磨炼开发技能。
- 若本书激起一丝兴趣，请**追随突如其来的灵感，试着做一款简单游戏**——新发现与乐趣必在其中。

## 全书方法论的回响

结语把全书各章串联成一个可操作的闭环：

| 全书主题 | 对应行动 |
|---|---|
| 小游戏哲学（第1章） | 别等"准备好"，先做一款 10 分钟可玩的 |
| 设备与环境（第2章） | 选浏览器 + 一个 JS 库，今天就开工 |
| 创意构思（第3章） | 卡住就用 SCAMPER / 机制拆解重组 |
| 约束（第4章） | 主动设约束（单按钮/限时）激发创意 |
| 难度（第5章） | 用 sqrt/锯齿公式让难度有起伏 |
| Juiciness（第6章） | 加粒子/震屏/音效提升手感 |
| 工具（第7章） | 自建轻量库（crisp-game-lib）加速迭代 |
| 程序化生成（第8章） | 用自动化降低重复劳动 |
| AI 生成（第9章） | 拥抱 LLM 作为创作伙伴 |

## 核心论点

作者 keita 全书的潜台词：**小游戏的价值不在成品，而在"反复做、反复发、反复迭代"的过程**。350+ 作品不是炫耀，而是这个循环的自然积累——每一次尝试都让下一次更快、更好、更有趣。比起追求一款"完美大作"，持续产出小作品能带来更稳的进步与更持久的创作乐趣。
