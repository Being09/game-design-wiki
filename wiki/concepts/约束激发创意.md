# 约束激发创意

> 跨第4章综合。原书见 [第4章](../references/第4章-约束.md)。

## 定义

**约束作为创意催化剂**：预先给自己施加某些约束，再思考如何在约束内把游戏做有趣——这能催生区别于以往的创意，也能提升点子独特性。同时，标准化"做什么、怎么做"的约束能减少开发负担、加速完成。

## 约束类型矩阵（《Different Game Jam Formats》分类）

| 类型 | 说明 | 作用 |
|---|---|---|
| **Tools** | 指定可用引擎 | 促完成 |
| **Assets** | 预先提供图文素材 | 促完成 |
| **Visuals** | 限制分辨率/颜色数 | **促创意**（16x16、1D 场地、甚至 1x1 两色） |
| **Goals** | 定义目标 | **促创意**（非常规目标："毁灭自身""牺牲""利用敌人获胜"） |
| **Interaction** | 限制交互（"仅两按钮""仅鼠标""全键盘"） | **促创意**（核心，见单按钮） |
| **Genre** | 指定类型 | 促完成 |
| **Time Limit** | 限时完成 | 促完成 |

> **促创意**：interaction / visuals / goals；**促完成**：tools / assets / genre / time。

## 代表性约束实践

### 交互约束：单按钮（详见 [[单按钮游戏设计]]）
仅一按钮完成所有交互，迫使设计"瞬移/分裂/选择/属性反转"等独特机制。

### 视觉约束：极低分辨率
16x16 像素、1D 场地（一维，仅左右/上下）、1x1 两色显示——极度约束倒逼创意解法。

### 目标约束：非常规目标
"毁灭自身""牺牲""利用敌人获胜"等（来自 Game Jam Theme Generator）能孕育新体验。

### 尺寸约束：代码 golf（js13kGames / jsgame256）
- **js13kGames**：13KB 内做浏览器游戏。
- **jsgame256**：每帧 update 函数 ≤256 字符 + p5.js/Tone.js 库 + 单字符变量别名。
- **精髓**：信息密集之美——用最少组件达最大效果。
- **代价**：实现复杂动作很快超字符数，做出来 270 字符被迫砍机制很挫败。

## 核心论点

约束是双刃剑：
- **适度的、契合个人 sensibility 的约束** → 激发创意 + 加速完成。
- **过严的约束** → 难以为继，成枷锁。

若想多做游戏，**建立自己的创作规则集、标准化流程**是稳健策略。尺寸约束尤其只推荐给"觉得克服尺寸限制本身有趣"的人。

## 应用

- **选对约束类型**：想要新点子 → 用 interaction/visuals/goals 约束；想快速完成 → 用 tools/assets/genre/time 约束。
- **用约束生成器**：Game Jam Theme Generator / 专用创意生成器辅助施加约束。
- **标准化创作流水线**：固定工具、固定类型、固定流程，把精力集中在核心机制创新上。
