# 矩阵与变换

> 综合多本书的跨书主题文章（主要来源：《游戏中的数学与物理学》）。

## 定义

**矩阵与变换**是用矩阵统一表示并复合平移、旋转、缩放等空间变换的数学工具。它是 2D/3D 图形与运动的核心——角色的移动旋转、相机的视角变换、骨骼动画的层级关系，都建立在变换矩阵之上。四元数作为旋转的紧凑表示，解决了欧拉角的万向锁问题。

## 多本书视角对比

- **《游戏中的数学与物理学》**（[参考](../references/游戏中的数学与物理学（第2版）.md)）：第 4 部分（3D 数学）系统讲解。转换操作（位置/旋转/缩放 → 转换矩阵）、利用转换构建运动、插值、**四元数**（避免万向锁、便于插值）、**父子转换层级**（scene graph，构建复杂对象与骨骼动画）。强调矩阵是统一变换语言。
- **《代码本色》**（[参考](../references/代码本色.md)）：Shiffman 主要在 2D Processing 中工作，矩阵变换着墨较少（旋转用角度+三角函数即可）。偏"2D 实践"。

## 方法/模型

- **变换矩阵**：平移、旋转、缩放统一为矩阵，可复合（矩阵乘法）。
- **四元数（Quaternion）**：四元组表示三维旋转，避免万向锁，便于球面插值（slerp）。
- **父子转换层级**：子对象继承父对象的变换，构建骨骼、车辆等复合结构（场景图）。
- **插值**：线性插值（位置）、球面插值（旋转，四元数 slerp）。
- **齐次坐标**：用 (x,y,z,w) 统一表示点与方向，使平移也能用矩阵乘法。

## 具体例子

**2D 旋转矩阵**（绕原点转 θ）：

```
| cos θ  -sin θ |
| sin θ   cos θ |
```

点 (x,y) 旋转后 = 矩阵 × (x,y)ᵀ。把旋转、缩放、平移各做成 3×3 齐次矩阵，再相乘就得到复合变换——这就是为什么所有游戏引擎都用矩阵而非"角度+三角函数"算运动：复合只需一次矩阵乘。

**变换顺序至关重要**：`先平移后旋转` ≠ `先旋转后平移`。绕原点转 90° 再平移，和平移后再绕原点转，结果完全不同。矩阵乘法不满足交换律（M₁M₂ ≠ M₂M₁），所以引擎里变换有固定顺序（通常 缩放→旋转→平移，即 SRT）。

**齐次坐标为何必要**：线性变换（旋转/缩放）用 2×2 矩阵即可，但平移是加法不是乘法，无法用 2×2 表示。升到 3 维齐次坐标 (x,y,1) 后，平移变成矩阵乘法：

```
| 1 0 tx |   | x |   | x+tx |
| 0 1 ty | × | y | = | y+ty |
| 0 0 1  |   | 1 |   |  1   |
```

这样平移/旋转/缩放都能用同一套矩阵乘法复合，是整个图形管线的基础。

**万向锁（Gimbal Lock）与四元数**：欧拉角（俯仰/偏航/翻滚）表示旋转时，当俯仰到 ±90°，偏航轴和翻滚轴重合，丢失一个自由度——相机突然抽搐翻转。四元数 q = (w, x, y, z) 用四元组表示旋转，无奇异点，且 slerp 球面插值能让旋转平滑过渡。3D 相机/骨骼几乎都用四元数。

**父子层级的应用**：手臂 = 肩膀(父) → 上臂 → 肘 → 前臂 → 手。手的世界位置 = 肩变换 × 肘变换 × 腕变换 × 局部手位置。转肩膀时整条手臂跟着动——这就是骨骼动画和载具（车身→轮子）的原理。

## 应用

- 3D 角色/相机的移动旋转缩放用变换矩阵。
- 骨骼动画用父子层级 + 四元数旋转。
- 相机平滑跟随用插值。
- 3D 渲染管线用矩阵完成模型→世界→视图→投影变换。
