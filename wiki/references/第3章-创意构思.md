# 第3章 小游戏创意构思法

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

创意阶段既兴奋又费力。好点子推进顺利，平庸点子举步维艰——但小游戏的好处是**点子不行直接丢弃**，试错成本极低。本章给两条路径：通用构思法（借鉴外部智慧）+ 机制拆解重组法（作者自创的标签化方法）。

## 路径一：通用构思法

### Reddit r/gamedev 社区四法
1. 从喜爱的现有游戏汲取灵感。
2. 融合看似奇怪的元素（卡牌+钓鱼、塔防+Minecraft、Pac-Man 做 FPS）。
3. 分析其他游戏，思考"如果是你会怎么做 differently"。
4. 把创意收敛到"想做的"与"能做的"交集。
5. 拥抱限制，剥离复杂性创造新东西。

### GameMaker "14 法"摘录（适合小游戏的）
- 玩经典老游戏，思考如何扩展现代化其基础（如超级马里奥之于 2D 平台跳跃）。
- 凭空想概念难，先选个类型再围绕它建构。
- 选类型难就先选角色，用角色特质展开世界。
- 用思维导图从核心（类型/主角）发散关键词。
- 灵感随时降临，手边常备记事本。
- **SCAMPER 法**：对现有游戏做替代/组合/增添/修改/消除/重排/他用/反转。
- 从影视、音乐、书籍、传统艺术、现实世界、个人经历取材。
- 用游戏创意生成器；参加主题 Game Jam；逛游戏社区直接问玩家。

### 作者个人实践（14 条博客方法摘录）
- 借鉴现有游戏（含传统体育）的母题与规则；把老游戏的一部分重制成独立游戏。
- 融入连锁反应、重力等惯用机制；以"集体收集/击败"的高分作为乐趣基础。
- 从自然现象/几何移植有趣行为；把物理行为与操控绑定产生乐趣。
- 视觉化呈现成功操作以增强成就感；基于单按钮等约束设计交互。
- **始终考虑风险与回报的平衡**；从呈现给玩家的风险出发设计新机制（risk-driven）。
- 平衡爽快感与紧张感；寻找提升游戏节奏的规则；用设定/背景连接游戏与叙事；移除令玩家困惑的新奇以降低门槛。

### 专用创意生成器
通用老虎机式生成器（letsmakeagame.net）输出太泛，作者做了**小游戏专用生成器**（mini-game-idea-generator），产出"偏转导弹""战斗的屋顶""放大电流"等具体点子。

## 路径二：机制拆解重组法（作者核心方法）

把自制游戏拆解为 **机制标签**，重组生成新游戏。作者维护了 [action-mini-game-mechanic-tags](https://github.com/abagames/action-mini-game-mechanic-tags)，按维度分类：

| 维度 | 标签示例 |
|---|---|
| **player（玩家）** | 旋转、往复、反射... |
| **weapon（武器）** | 抛物线飞行（投掷方式/投掷物/落地效果各异） |
| **obstacle（障碍）** | 混入惩罚道具（迫使观察时机而非盲目连击） |
| **field（场地）** | 1D 场地（一维，仅左右/上下移动） |
| **rule（规则）** | 近距奖励（在障碍旁放奖励点=风险回报雏形）、按键属性反转 |
| **on_pressed（按键）** | 反转属性/状态 |

### 游戏机制的定义
玩家可执行的有组织动作 + 这些动作触发的事件。用于分析游戏设计或构思新游戏。**gameplay 是游戏机制间的流转**。粒度无标准：可抽象到回合/资源管理/风险回报，也可细到跳跃/格挡/旋转/强化。作者为构思动作小游戏，聚焦**较细的局内动作**，强调看实际例子、理解机制如何组合、识别给玩家的风险（挑战）与回报。

### 使用流程
1. 看标签列表页 → 理解可整合进动作小游戏的机制。
2. 看标签详情页 → 考察用该机制的其他游戏。
3. 任意组合多个标签 → 用这些机制构思游戏。

> [!tip] 偏见说明
> 因用自制游戏做例子，内容偏向动作小游戏（尤其单按钮游戏），但作为构思范例足矣。

## 核心论点

构思方法很多，前辈智慧能加速，但**最佳方式是创造性思考 + 多产**。小游戏的优势是构思与实现都极快，点子失败就丢，轻松尝试。
