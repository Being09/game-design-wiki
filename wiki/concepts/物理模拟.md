# 物理模拟

> 综合多本书的跨书主题文章。

## 定义

**物理模拟**是用代码重现现实世界的运动与力学——力如何产生加速度、物体如何碰撞反弹、弹簧如何振荡、粒子如何飘散。它是游戏手感、自然感动画与衍生艺术的根基。核心是用数值积分（如欧拉法）逐帧推进运动，把连续的物理方程离散化为可计算的帧间更新。

## 多本书视角对比

- **《代码本色》**（[参考](../references/代码本色.md)）：Shiffman 从"自然感"出发，用 Processing 教如何用代码模拟自然系统。以向量与牛顿第二定律为地基（`位置 += 速度; 速度 += 加速度; 加速度 = 合力/质量`），逐类建模重力、摩擦力、流体阻力、万有引力，并引入 Box2D（碰撞/刚体/关节）与 toxiclibs（粒子/弹簧）物理库。强调"贴近真实的移动"而非物理精确，配合粒子系统、转向力、群集等扩展。
- **《游戏中的数学与物理学》**（[参考](../references/游戏中的数学与物理学（第2版）.md)）：Flynt & Kodicek 从程序员视角，系统讲解运动学方程、牛顿三定律、冲量、能量/动量守恒、摩擦力、角运动、简谐振荡，以及碰撞检测与处理流程。更偏"精确建模与数值方法"，含 ODE 数值求解（划界法、梯度法）。

两书互补：代码本色偏"自然感模拟 + 实践项目"，数学与物理学偏"精确建模 + 数值方法"。前者教怎么写出动的代码，后者教背后的数学严谨性。

## 方法/模型

- **运动积分**：欧拉法 `位置 += 速度; 速度 += 加速度`，是逐帧推进的基础。
- **牛顿第二定律**：`F = ma`，所有力建模为向量，累加得合力，除以质量得加速度。
- **力的建模流程**：把力拆成方向（单位向量）与大小（标量），再翻译成向量。
- **碰撞处理流程**：判定 → 求最早碰撞时间 → 计算碰撞点与法线 → 解算响应（弹性/非弹性）。
- **物理引擎集成**：Box2D（碰撞/刚体/关节）vs toxiclibs（粒子/弹簧/Verlet），关键在坐标转换与对象映射。
- **数值方法**：对无法解析的 ODE 用划界法、梯度法等近似积分。

## 具体例子

**逐帧物理积分循环**（每个游戏物理对象的核心 update）：

```
// 牛顿第二定律：合力 → 加速度
acceleration = sum(forces) / mass

// 数值积分（显式欧拉）
velocity += acceleration * dt
position += velocity * dt

// 清空本帧力，等下一帧重新累加
forces.clear()
```

重力就是每帧往 forces 加一个向下常量 `(0, 9.8 * mass)`——持续累加产生抛物线轨迹，无需手写抛物线公式。

**力的建模流程**（万有引力实例）：

```
dir = (other.position - this.position).normalize()  # 方向（单位向量）
d = distance(this, other)
magnitude = G * mass1 * mass2 / (d * d)             # 大小（标量，平方反比）
force = dir * magnitude                              # 翻译成向量
this.applyForce(force)
```

任何力都遵循"方向（单位化）× 大小（标量公式）"的拆解，让力建模可组合、可叠加。

**欧拉法的精度问题与改进**：显式欧拉 `v += a·dt; x += v·dt` 在高速或大 dt 时能量会"泄漏"（弹簧越振越大）或爆炸。改进：
- **半隐式（辛）欧拉**：先更新速度再加到位置 `v += a·dt; x += v·dt`——对保守系统稳定，游戏最常用。
- **Verlet 积分**：用当前位置与上一位置推速度，省存储且稳定，适合粒子/布料。
- **RK4（四阶龙格-库塔）**：精度高但 4 倍计算量，仅精密物理用。

游戏物理的黄金法则：**稳定性 > 精确性**。半隐式欧拉够用且快，宁可微小误差也不要数值爆炸。

**弹簧-质点系统（Verlet，布料/绳索基础）**：每个点记录当前位置和上一位置，速度隐含在差值里：

```
temp = position
position += (position - prevPosition) * (1 - friction) + acceleration * dt²
prevPosition = temp
```

再加约束（两点距离固定）反复迭代修正——能稳定模拟绳索、布料、软体，是 toxiclibs/Verlet 物理的核心。

**碰撞响应的能量**：恢复系数 e∈[0,1] 控制弹性。e=1 完全弹性（台球，能量守恒）；e=0 完全非弹性（泥团，粘在一起）。游戏里角色撞墙 e≈0（停下），皮球 e≈0.8（弹起但有损耗），调 e 即可控制"手感"。

## 应用

- 角色运动、抛射物、车辆物理用运动积分 + 力的累加。
- 大量粒子（火焰/烟雾/爆炸）用粒子系统 + 短生命周期管理。
- 碰撞、关节、布料、绳索用物理引擎（Box2D/toxiclibs/Verlet）。
- 弹簧、钟摆、波用简谐运动方程。
