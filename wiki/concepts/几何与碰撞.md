# 几何与碰撞

> 综合多本书的跨书主题文章（主要来源：《游戏中的数学与物理学》）。

## 定义

**几何与碰撞**是判定游戏对象是否相交、在哪里相交、如何响应的算法集合。它从简单形状（圆、矩形）的相交判定，到复杂形状（Bezier 曲线、样条、凸多边形）的精确碰撞，再到包围体层次结构（BVH/AABB）的加速剔除，是动作、射击、物理类游戏的核心系统。

## 多本书视角对比

- **《游戏中的数学与物理学》**（[参考](../references/游戏中的数学与物理学（第2版）.md)）：第 2、4 部分系统讲解。简单形状碰撞（圆-圆、圆-墙、矩形轴对齐/带角度、椭圆、圆-矩形）、碰撞处理（单一/多次、弹性/非弹性）、复杂形状（参数函数、Bezier、样条、凸凹判定、点在形状内）、3D 碰撞（球体/椭球/盒/柱）、加速方案（四叉树/八叉树、BSP、BVH/AABB）。
- **《代码本色》**（[参考](../references/代码本色.md)）：碰撞主要通过物理库（Box2D）处理，不自写碰撞算法；强调用现成引擎而非重复造轮子。

## 方法/模型

- **简单形状碰撞**：圆-圆（距离 < 半径和）、矩形（轴对齐 AABB / 带角度 OBB）、圆-矩形。
- **碰撞处理流程**：判定 → 求最早碰撞时间 → 计算碰撞点与法线 → 解算响应（冲量法，弹性/非弹性）。
- **复杂形状**：Bezier 曲线、Catmull-Rom 样条、凸凹判定、点在形状内（射线法）。
- **3D 碰撞**：球体、椭球体、盒体、柱体；网格碰撞。
- **加速剔除**：四叉树/八叉树（空间划分）、BSP（二分空间）、BVH/AABB（包围体层次结构）。

## 具体公式与例子

**圆-圆碰撞**（最简单、最廉价，永远优先用圆近似）：

```
d = distance(A.center, B.center)
if (d < A.radius + B.radius)  // 重叠
    法线 = (B.center - A.center) / d   // 单位向量
```

**AABB 轴对齐矩形碰撞**（无旋转时极廉价，4 个区间比较）：

```
重叠X = (A.left < B.right) and (A.right > B.left)
重叠Y = (A.top < B.bottom) and (A.bottom > B.top)
碰撞 = 重叠X and 重叠Y
```

**连续碰撞（CCD）防穿透**：高速子弹逐帧 AABB 判定会"跳过"薄墙。改求最早碰撞时间 t∈[0,1]：

```
t = (墙边界 - 子弹起点) / 子弹速度
若 0 ≤ t ≤ 1 则本帧内某时刻会撞，按 t 处位置处理
```

**冲量法弹性碰撞响应**（沿法线交换动量分量）：

```
法线 n = 碰撞点单位法向量
相对速度沿法线分量 v_rel_n = (vB - vA) · n
if (v_rel_n > 0) return  // 已在分离，不处理
冲量 j = -(1+e)·v_rel_n / (1/mA + 1/mB)   // e=恢复系数，1=完全弹性
vA -= (j/mA) · n
vB += (j/mB) · n
```

**AABB vs OBB 取舍**：角色用 AABB（廉价），旋转的车/剑用 OBB（需 SAT 分离轴定理，更贵但精确）。策略：用 AABB 做粗判，命中后再用 OBB 精判，两全其美。

**空间划分加速**：1000 个子弹两两判定是 O(n²)=10⁶ 次。四叉树按区域分组，只判同区域对象，降到接近 O(n·log n)。弹幕游戏的命门就在这里。

## 应用

- 动作/射击游戏的命中判定用圆/矩形碰撞。
- 物理游戏的弹跳、堆叠用冲量法碰撞响应。
- 大量对象的碰撞（如弹幕）用空间划分加速。
- 复杂地形/角色用网格或包围盒近似。
