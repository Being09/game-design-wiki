# Utility 理论（Utility Theory）


## 概述

Utility 理论（效用理论）是一种基于评分的 AI 决策方法。每个行为计算一个"效用值"（utility score），AI 选择效用最高的行为执行。

与行为树的根本区别在于：

| 维度 | 行为树 | Utility 系统 |
|------|--------|-------------|
| 选择方式 | 硬逻辑（Success/Failure） | 软评分（0.0 ~ 1.0） |
| 行为优先级 | 静态（从左到右） | 动态（实时计算评分） |
| 可预测性 | 高（确定性强） | 低（行为自然多变） |
| 调参方式 | 调整树结构 | 调整曲线和权重 |
| 适用场景 | 确定性流程 | 灵活选择 |

系统介绍了 Utility 理论及其与行为树的融合方案，使之成为游戏AI中既灵活又可控的决策方案。

### 适用场景

- NPC 武器选择（根据距离/伤害/弹药选择）
- 动态行为优先级（饥饿/疲劳/社交需求的权衡）
- 战术决策（哪个掩体最好、哪个目标最威胁）
- 与行为树融合（行为树控制流程，Utility 控制选择）

## 核心概念

### 效用曲线（Utility Curves）

效用曲线将原始输入映射到 0-1 的评分值。介绍了多种曲线类型：

```
线性:     score = clamp(x / max, 0, 1)
指数:     score = pow(x / max, 2)
S形:      score = 1 / (1 + exp(-k * (x - midpoint)))
Logistic: score = 1 / (1 + exp(-k * x))
分段:     手动定义关键点之间的插值
```

**选型指南：**

| 曲线类型 | 行为模式 | 示例 |
|---------|---------|------|
| 线性 | 均匀增长 | 威胁度随距离增加 |
| 指数 | 低值时不敏感，高值时陡增 | 弹药不足时才考虑补给 |
| S形（Logistic） | 中间过渡平滑，两端饱和 | 根据血量决定畏缩程度 |
| 分段 | 设计师精确控制 | 特定距离区间内才感兴趣的感知 |

### 多因素加权（Weighted Scoring）

一个行为的最终效用值由多个"考虑因素"（considerations）加权求和：

```python
# 示例：攻击行为的效用评分
utility = (
    health_factor * 0.3 +    # 血量因素 (权重30%)
    distance_factor * 0.3 +  # 距离因素 (权重30%)
    ammo_factor * 0.2 +      # 弹药因素 (权重20%)
    threat_factor * 0.2      # 威胁因素 (权重20%)
)
```

**加权原则：**
- 所有权重之和 = 1.0
- 关键因素赋予高权重
- 权重可通过调试器实时调整

### 噪声注入（Noise Injection）

为避免 AI 行为过于确定（永远是最高分），可注入噪声：

```python
# 噪声注入：增加自然变化
def apply_noise(score: float, noise_level: float = 0.1) -> float:
    return score + random.uniform(-noise_level, noise_level)

# 或者使用高斯噪声，使小偏差更常见
def apply_gaussian_noise(score: float, std: float = 0.05) -> float:
    return score + random.gauss(0, std)
```

## 伪代码实现

```python
import random
from enum import Enum

class Consideration:
    """单个考虑因素 — 将原始输入映射为效用值"""
    def __init__(self, input_fn, curve_fn, weight: float):
        self.input_fn = input_fn    # 获取原始值（如距离、血量）
        self.curve_fn = curve_fn    # 曲线映射函数
        self.weight = weight        # 权重

    def score(self, bb) -> float:
        raw = self.input_fn(bb)
        return self.curve_fn(raw) * self.weight


class UtilityAction:
    """一个带效用评分的可选行为"""
    def __init__(self, name: str, considerations: list, action_fn):
        self.name = name
        self.considerations = considerations
        self.action_fn = action_fn

    def evaluate(self, bb) -> float:
        total = sum(c.score(bb) for c in self.considerations)
        return total


class UtilityAI:
    """Utility AI 决策系统"""
    def __init__(self):
        self.actions: list[UtilityAction] = []

    def add_action(self, action: UtilityAction):
        self.actions.append(action)

    def decide(self, bb, noise: float = 0.0):
        scores = []
        for action in self.actions:
            score = action.evaluate(bb)
            if noise > 0:
                score += random.uniform(-noise, noise)
            scores.append((score, action))

        # 选择最高评分的行为
        scores.sort(key=lambda x: x[0], reverse=True)
        return scores[0][1] if scores else None


# ---- 使用示例 ----

def health_curve(hp):
    """血量越低，需求越高的指数曲线"""
    return 1.0 - (hp / 100.0) ** 2  # 20%血 → 0.96, 80%血 → 0.36

def distance_curve(dist):
    """近处得分高，远处得分低的线性曲线"""
    return max(0, 1.0 - dist / 50.0)

def ammo_curve(ammo):
    """弹药越少越需要补给"""
    return max(0, 1.0 - ammo / 30.0)

# 构建决策系统
ai = UtilityAI()

ai.add_action(UtilityAction("攻击", [
    Consideration(lambda bb: bb.get("distance"), distance_curve, 0.4),
    Consideration(lambda bb: bb.get("health"), lambda h: h/100, 0.3),
    Consideration(lambda bb: bb.get("ammo"), lambda a: a/30, 0.3),
], lambda bb: print("攻击！")))

ai.add_action(UtilityAction("撤退", [
    Consideration(lambda bb: 100 - bb.get("health"), health_curve, 0.5),
    Consideration(lambda bb: bb.get("distance"), lambda d: d/50, 0.3),
    Consideration(lambda bb: bb.get("ammo"), ammo_curve, 0.2),
], lambda bb: print("撤退！")))

ai.add_action(UtilityAction("补给", [
    Consideration(lambda bb: 30 - bb.get("ammo"), lambda a: a/30, 0.5),
    Consideration(lambda bb: bb.get("health"), health_curve, 0.3),
    Consideration(lambda bb: bb.get("distance"), lambda d: 1-d/50, 0.2),
], lambda bb: print("补给！")))

# 执行
bb = {"health": 30, "distance": 15, "ammo": 5}
best = ai.decide(bb, noise=0.05)
best.action_fn(None)  # 可能输出：补给！
```

## 实战要点

### 行为树 + Utility 融合方案

这是最实用的模式：行为树控制**流程和结构**，Utility 控制**具体选择**。

```
行为树顶层：
  Selector
  ├── Sequence: [条件: 需要治疗? → 动作: 寻找治疗包]
  │                              └── 内部用 Utility 选哪个治疗包
  ├── Sequence: [条件: 有敌人? → 动作: 战斗]
  │                              └── 内部用 Utility 选武器/战术
  └── Sequence: [动作: 巡逻]
                  └── 内部用 Utility 选巡逻路线
```

**优势：**
- 保留了行为树的流程控制能力
- 获得了 Utility 的灵活选择能力
- 行为树的"条件"做关卡，Utility 做"选择"

### 曲线调参技巧

1. **从线性开始**：先保证逻辑正确，再调整曲线形状
2. **使用可视化工具**：绘制曲线实时预览
3. **关注过渡点**：曲线在什么输入值下产生行为切换
4. **权重 vs 曲线**：权重控制因素间的重要性，曲线控制单个因素的行为变化模式

### 性能考量

- Utility 评分可在多个帧间分布计算（分摊开销）
- 缓存近期评分结果（环境变化不大时可复用）
- 使用 LOD：远距离 AI 简化考虑因素数量

## 反模式（NEVER 列表）

| # | 反模式 | 原因 |
|---|--------|------|
| 1 | ❌ 效用值未归一化 | 各考虑因素的输出必须在相似范围，否则高值因素主导 |
| 2 | ❌ 过多因素的加权求和 | 超过 5-7 个因素会导致"平均化"，关键因素被稀释 |
| 3 | ❌ 权重靠直觉设置 | 权重应该在游戏中实测调优 |
| 4 | ❌ 没有噪声差异 | 所有AI加相同噪声 → 行为仍然趋同 |
| 5 | ❌ 替代行为树的一切 | Utility 不适合流程型任务（如"开门→进房间→检查"） |
