# 向量与运动

> 综合多本书的跨书主题文章。

## 定义

**向量**是既有大小又有方向的数学对象，是描述运动（位移、速度、加速度、力）的基本数据单元。把成对的标量（x/y、xspeed/yspeed）统一为向量对象后，运动模拟就从分散的坐标运算变成统一的向量运算，大幅简化代码并提升表达力。

## 多本书视角对比

- **《代码本色》**（[参考](../references/代码本色.md)）：Shiffman 把向量作为全书第 1 章地基。用 PVector 对象统一位置/速度/加速度，覆盖加减、数乘、长度、单位化，并建立 `位置 += 速度; 速度 += 加速度` 的运动积分模型。强调向量是把"分散的 xspeed/yspeed"升级为可统一运算的对象。
- **《游戏中的数学与物理学》**（[参考](../references/游戏中的数学与物理学（第2版）.md)）：Flynt & Kodicek 给出更严谨的数学定义——"指令向量"概念、点积（标量积，求夹角/分量/正交）、叉积（三维法向/转矩）、向量方程描述形状与路径。覆盖更广（含三维叉积、齐次坐标）。

两书一致认为向量是运动/几何建模的核心；代码本色偏"编程实践"，数学与物理学偏"数学严谨 + 三维扩展"。

## 方法/模型

- **向量运算**：加减、数乘、长度（模）、单位化（normalize）。
- **点积（Dot Product）**：`u·v = |u||v|cos α`，求夹角、投影分量、判断正交/同向，用于视野判断、光照。
- **叉积（Cross Product）**：三维向量，结果垂直于两向量，求法线、转矩、面积。
- **运动积分**：`位置 += 速度; 速度 += 加速度`，欧拉法逐帧推进。
- **单位向量定方向**：先单位化得方向，再乘标量得大小。

## 具体例子

**单位化定方向**（游戏最常用模式）：让 A 朝向 B 运动——

```
direction = (B - A)          # 从 A 指向 B 的向量
direction.normalize()        # 单位化：去掉距离，只留方向
A.position += direction * maxSpeed   # 按最大速度前进
```

不单位化的话，B 越远 A 跑得越快（直接加 (B-A) 会把距离当速度），完全失控。

**点积判视野**：玩家在 P 朝向 D（单位向量），判断敌人 E 是否在前方扇形视野：

```
toEnemy = (E - P).normalize()
cosAngle = D · toEnemy        # = cos 夹角
if (cosAngle > 0.5)  // cos60° ≈ 0.5
    敌人在 120° 前方视野内
```

点积符号直接给方向关系：>0 同向、=0 垂直、<0 反向，无需调用昂贵的 acos。

**运动积分的物理含义**：每帧 `位置 += 速度`（位移 = 速度×Δt，Δt=1 帧）；`速度 += 加速度`（速度变化 = 加速度×Δt）。重力就是恒定向下加速度，速度持续累加产生自由落体的抛物线轨迹——这就是为什么"往下加个常数"就能模拟重力。

**欧拉法的局限**：逐帧累加误差积累，高速运动或大 Δt 会让轨迹偏离真实积分。高速物体用半隐式欧拉（先更新速度再加到位置）或 RK4 更稳定。

## 应用

- 角色移动、抛射物轨迹用位置/速度/加速度向量。
- 视野检测、光照计算用点积。
- 3D 法线、碰撞法线用叉积。
- 路径跟随、流场用向量场。
