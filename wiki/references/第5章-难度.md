# 第5章 小游戏的适度难度

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

太难令人挫败，太易令人无聊。小游戏常用手法是**随时间逐步提升难度**，直到超过玩家技能，失误即 game over。本章给出"适度"的判断标准、连续难度曲线公式、以及多关卡难度设定法。

## 适度难度的判断标准

依据 **Csikszentmihalyi 心流模型（Flow）**：挑战与技能平衡时进入沉浸。

**关键判据**：难度提升、玩家失误时——
- 若玩家认为**是自己技术不足** → 难度适当。
- 若玩家遭遇**不可规避的攻击**（感到不公） → 难度不当。

## 调难度的方法

| 方法 | 优劣 |
|---|---|
| **游戏速度**（自 Game & Watch 起） | **首选**——玩家与敌人速度同比例提升，避免"早期能躲的敌人后期躲不掉"的不公。 |
| **增加敌人数量** | 易单调，需配合。 |
| **放大障碍物** | 风险高——障碍大到玩家跳跃能力无法规避时产生不公。 |
| **引入新元素**（新敌人/武器/地形/道具，要求高级技巧） | 理想但小游戏实现成本高。 |

> 多种方法组合，游戏更丰富，玩家不易腻。

## 难度曲线（连续递进）

**核心公式**（假设游戏 3 分钟、难度翻倍时多数人出局）：

```
[Difficulty] = sqrt([Elapsed frame count] * 0.0001) + 1
```

- `frame`：1/60 秒。3 分钟 = 10800 帧，代入得难度 2.04（≈翻倍）。
- 用 `sqrt` 而非线性，让难度在一段时间后**上升更平缓**。

**但单一公式不普适**——有时快速提升更紧张。作者做了 **diffi-tween** 可视化调参工具：右侧 UI 调曲线，左侧立即在"躲落石"游戏试玩。

### 曲线维度

每条曲线由**公式类型** + **上升率（climb）**定义，可叠加**锯齿波（saw）**——固定间隔让难度骤降，引入变化（Game & Watch 也用此法）。

### 多参数调参策略（以落石为例）

- 落石**速度**：线性提升维持紧张，偶尔锯齿波放缓。
- 落石**大小**：用 sqrt 平缓提升，缓解不公感（大石头给人"无法规避"的不公印象）。
- 早期太易无聊 → 引入"贴近石头得奖励分"的风险回报机制，从早期就制造紧张。

> 多参数时调参更难，需用不同曲线分别对待不同参数。详见 [[难度曲线设计]]。

## 基于关卡的难度（离散递进）

适用于有多关卡的游戏。**避免单调线性递增**，用公式制造有波动的难度：

```
difficulty = pow(random(), 100 / ([当前关卡数] + 1))
```

- 横轴 `random()`（0-1），纵轴难度（0 最易、1 最难）。
- 关卡数低时，难度多贴在低端；关卡数高时，难度向上扩展 → 高关卡大概率更难。
- 但因 random，偶尔某关比其序号暗示的明显更易/更难。

**多参数应用**：敌人数/射击频率/子弹速度各用此公式 + random → 出现"敌多但少射击""一敌但快弹"等组合变化。

**缺点**：后期所有参数贴顶（=1），无法无限提升；在封顶状态定难度很难。

## 核心论点

无论连续还是离散，**关键是难度提升不能单调**——通过制造玩家体验到的紧张波动，获得更愉悦的推进。
