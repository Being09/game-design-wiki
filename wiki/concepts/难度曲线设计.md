# 难度曲线设计

> 跨第5章综合。原书见 [第5章](../references/第5章-难度.md)。

## 定义

小游戏难度设计的核心目标：依据 **Csikszentmihalyi 心流模型**让挑战匹配玩家技能，避免太易无聊、太难挫败。作者提供两大范式——**连续难度曲线**（无关卡，随时间递进）与**离散关卡难度**（多关卡，每关独立定难度）。

## 适度难度的判断标准

难度提升、玩家失误时：
- 玩家**归因于自己技术不足** → 难度适当。
- 玩家遭遇**不可规避攻击**（感到不公） → 难度不当。

## 范式一：连续难度曲线

### 核心公式（假设游戏3分钟、难度翻倍时多数人出局）
```
[Difficulty] = sqrt([Elapsed frame count] * 0.0001) + 1
```
- `frame` = 1/60 秒；3 分钟 = 10800 帧 → 难度 2.04（≈翻倍）。
- 用 `sqrt` 而非线性，让难度一段时间后**上升更平缓**。

### 曲线维度
每条曲线由**公式类型**（sqrt / linear / pow）+ **上升率（climb）**定义，可叠加**锯齿波（saw）**——固定间隔让难度骤降引入变化（Game & Watch 也用）。

### 多参数调参策略（以落石为例）
不同参数用不同曲线，避免单一参数导致不公：
- 落石**速度**：线性提升维持紧张，偶尔锯齿波放缓。
- 落石**大小**：sqrt 平缓提升（大石头给人"无法规避"的不公感）。
- 早期太易 → 引入**风险回报**（贴近石头得奖励分）从早期就制造紧张。

**工具**：作者 [diffi-tween](https://abagames.github.io/diffi-tween/) 可视化调曲线 + 即时试玩。

## 范式二：离散关卡难度

多关卡游戏用公式制造**有波动的难度递进**（避免单调线性）：
```
difficulty = pow(random(), 100 / ([当前关卡数] + 1))
```
- 横轴 `random()`（0-1），纵轴难度（0 最易、1 最难）。
- 关卡数低 → 难度多贴低端；关卡数高 → 难度向上扩展（高关卡大概率更难）。
- 因 random，偶尔某关比序号暗示的明显更易/更难。

**多参数应用**：敌人数/射击频率/子弹速度各用此公式 + random → "敌多但少射击""一敌但快弹"等组合变化。

**缺点**：后期所有参数贴顶（=1），无法无限提升；封顶状态定难度很难。

## 调难度的方法对比

| 方法 | 优劣 |
|---|---|
| **游戏速度**（自 Game & Watch） | **首选**——玩家与敌人同比例提速，避免"早期能躲后期躲不掉"的不公 |
| 增加敌人数量 | 易单调 |
| 放大障碍物 | 风险高——大到无法规避时产生不公 |
| 引入新元素（新敌人/武器/地形） | 理想但小游戏实现成本高 |

## 应用

- **无关卡街机风游戏**：用 sqrt 连续曲线 + 锯齿波，配合风险回报机制。
- **有关卡游戏**：用 pow 关卡公式，对多参数分别套用以制造组合变化。
- **调参原则**：单参数优先用游戏速度（最不易不公）；多参数分别用不同曲线类型；始终给玩家留"归因于技术"的出路。

## 核心论点

无论连续还是离散，**关键是难度提升不能单调**——通过制造玩家体验到的紧张波动（锯齿波、random 波动、风险回报），获得更愉悦的推进。
