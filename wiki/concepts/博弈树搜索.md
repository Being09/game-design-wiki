# 游戏博弈树搜索



## 概述

博弈树搜索是游戏AI中与"行为树"并列的另一大支柱技术。前者解决"怎么做"，后者解决"怎么想"——即在规则允许的范围内，通过前瞻搜索找到最优动作序列。

本技能用四章篇幅系统介绍了游戏搜索技术：从经典搜索到移动平台的优化，再到前沿的MCTS。

### 适用场景

- 回合制策略游戏的AI对手
- 棋类游戏（象棋、围棋、黑白棋）
- 卡牌游戏的出牌决策
- 实时策略中的战术搜索
- 任何需要"想几步之后"的场景

## 博弈搜索基础

### 搜索问题的形式化

一个游戏搜索问题由三要素定义：

```
游戏搜索三要素：
├── 状态空间（State Space）— 所有可能的游戏局面
├── 动作集合（Action Set）— 每个状态下可做的操作
└── 终止条件（Terminal Condition）— 游戏何时结束
```

### Minimax 算法

```python
class Minimax:
    """经典极小化极大搜索"""
    def search(self, state, depth, maximizing_player):
        if depth == 0 or state.is_terminal():
            return state.evaluate(), None

        if maximizing_player:
            best_score = -float('inf')
            best_action = None
            for action in state.get_legal_actions():
                next_state = state.apply(action)
                score, _ = self.search(next_state, depth - 1, False)
                if score > best_score:
                    best_score = score
                    best_action = action
            return best_score, best_action
        else:
            best_score = float('inf')
            best_action = None
            for action in state.get_legal_actions():
                next_state = state.apply(action)
                score, _ = self.search(next_state, depth - 1, True)
                if score < best_score:
                    best_score = score
                    best_action = action
            return best_score, best_action
```

## Personality Reinforced Search

**专为移动策略游戏设计的搜索算法。** 核心思想：搜索不仅考虑"最优步"，还要考虑"符合AI个性的步"。

### 个性强化搜索

```python
class PersonalityReinforcedSearch:
    """个性强化搜索：平衡最优性和个性"""
    def __init__(self, personality_traits):
        self.traits = personality_traits
        # traits 示例:
        #   { "aggression": 0.8, "risk_tolerance": 0.3, "creativity": 0.6 }

    def evaluate_with_personality(self, state, player):
        """带个性偏好的评估函数"""
        base_score = state.evaluate(player)

        # 个性修正
        aggression_bonus = self._aggression_score(state, player)
        risk_penalty = self._risk_penalty(state, player)
        creativity_bonus = self._creativity_bonus(state, player)

        personality_score = (
            aggression_bonus * self.traits["aggression"] +
            creativity_bonus * self.traits["creativity"]
        ) - risk_penalty * self.traits["risk_tolerance"]

        return base_score + personality_score * 0.3  # 个性占30%权重

    def search(self, state, depth, alpha, beta, player):
        """带个性的搜索"""
        if depth == 0 or state.is_terminal():
            return self.evaluate_with_personality(state, player), None

        actions = state.get_legal_actions()
        # 按个性排序动作（进攻型AI优先考虑攻击动作）
        actions = self._sort_by_personality(actions, state, player)

        best_score = -float('inf')
        best_action = None
        for action in actions:
            next_state = state.apply(action)
            score, _ = self.search(next_state, depth - 1, alpha, beta, player)
            if score > best_score:
                best_score = score
                best_action = action
            alpha = max(alpha, best_score)
            if beta <= alpha:
                break
        return best_score, best_action
```

**个性参数的效果：**

| 个性维度 | 高值表现 | 低值表现 |
|---------|---------|---------|
| 进攻性（Aggression） | 偏好攻击、推进 | 偏好防守、发育 |
| 风险容忍（Risk Tolerance） | 愿意冒险搏收益 | 选择稳妥方案 |
| 创造力（Creativity） | 尝试非主流策略 | 使用标准策略 |

## Interest Search

**Interest Search——一种加速Minimax的算法。** 核心洞察：不是所有分支都值得深入搜索。

### 核心思想

传统Alpha-Beta对所有分支一视同仁。Interest Search只对"有趣"的分支深入搜索：

```python
class InterestSearch:
    """兴趣搜索——只深入搜索有趣的分支"""
    def __init__(self, max_depth=8, interest_threshold=0.3):
        self.max_depth = max_depth
        self.interest_threshold = interest_threshold

    def search(self, state, depth, alpha, beta, maximizing):
        if depth == 0 or state.is_terminal():
            return state.evaluate(), None, 0  # score, action, interest

        actions = state.get_legal_actions()
        # 第一步：浅层评估所有动作的"兴趣度"
        shallow_evals = []
        for action in actions:
            next_state = state.apply(action)
            # 浅层搜索（depth=1或2）评估这个分支的潜力
            interest = self._calculate_interest(next_state, depth)
            shallow_evals.append((action, interest))

        # 按兴趣度排序
        shallow_evals.sort(key=lambda x: x[1], reverse=True)

        best_score = -float('inf')
        best_action = None

        for action, interest in shallow_evals:
            # 只对兴趣度高于阈值的分支深入搜索
            if interest < self.interest_threshold and depth < self.max_depth - 2:
                # 不感兴趣的分支：浅层搜索
                next_state = state.apply(action)
                score, _ = self._shallow_search(next_state, depth - 2, not maximizing)
            else:
                # 感兴趣的分支：全深度搜索
                next_state = state.apply(action)
                score, _ = self.search(next_state, depth - 1, alpha, beta, not maximizing)

            if score > best_score:
                best_score = score
                best_action = action
            alpha = max(alpha, best_score)
            if beta <= alpha:
                break

        return best_score, best_action

    def _calculate_interest(self, state, depth):
        """计算一个状态有多"有趣"——变化大、动作多、有战术机会"""
        interest = 0.0
        if state.has_captures():
            interest += 0.4
        if state.has_threat():
            interest += 0.3
        if len(state.get_legal_actions()) > 10:
            interest += 0.2
        if state.is_near_terminal():
            interest += 0.3
        return min(1.0, interest)
```

**性能提升：** 相比标准Alpha-Beta，Interest Search 可在相同时间内搜索深 2-4 层。

## MCTS 及相关算法

**蒙特卡洛树搜索（Monte Carlo Tree Search）——不用评估函数的搜索方法。**

### MCTS 四阶段

```
┌──────────────────────────────────────────────┐
│                 MCTS 循环                      │
│                                              │
│  重复 N 次：                                  │
│  ┌─────────┐    ┌──────────┐                 │
│  │ 选择     │ →  │ 扩展     │                 │
│  │ Selection│    │ Expansion│                 │
│  └────┬────┘    └────┬─────┘                 │
│       │              │                       │
│       └──────────────┘                       │
│              │                               │
│              ▼                               │
│  ┌─────────┐    ┌──────────────┐             │
│  │ 回溯     │ ←  │ 模拟         │             │
│  │Backprop.│    │ Simulation   │             │
│  └─────────┘    └──────────────┘             │
└──────────────────────────────────────────────┘
```

```python
import math
import random

class MCTSNode:
    """MCTS 节点"""
    def __init__(self, state, parent=None, action=None):
        self.state = state
        self.parent = parent
        self.action = action
        self.children = []
        self.visits = 0
        self.wins = 0
        self.untried_actions = state.get_legal_actions()

    def ucb1(self, exploration_constant=1.41):
        """UCB1 置信上界"""
        if self.visits == 0:
            return float('inf')
        exploitation = self.wins / self.visits
        exploration = exploration_constant * math.sqrt(
            math.log(self.parent.visits) / self.visits
        )
        return exploitation + exploration


class MCTS:
    """蒙特卡洛树搜索"""
    def __init__(self, iterations=1000):
        self.iterations = iterations

    def search(self, root_state):
        root = MCTSNode(root_state)

        for _ in range(self.iterations):
            # 1. 选择
            node = self._select(root)
            # 2. 扩展
            if node.untried_actions:
                node = self._expand(node)
            # 3. 模拟
            result = self._simulate(node.state)
            # 4. 回溯
            self._backpropagate(node, result)

        # 选择访问次数最多的子节点
        best_child = max(root.children, key=lambda c: c.visits)
        return best_child.action

    def _select(self, node):
        """选择阶段：从根节点向下选择最有潜力的子节点"""
        while not node.untried_actions and node.children:
            node = max(node.children, key=lambda c: c.ucb1())
        return node

    def _expand(self, node):
        """扩展阶段：创建一个新的子节点"""
        action = node.untried_actions.pop()
        next_state = node.state.apply(action)
        child = MCTSNode(next_state, parent=node, action=action)
        node.children.append(child)
        return child

    def _simulate(self, state):
        """模拟阶段：随机玩到终局"""
        while not state.is_terminal():
            actions = state.get_legal_actions()
            action = random.choice(actions)
            state = state.apply(action)
        return state.get_result()

    def _backpropagate(self, node, result):
        """回溯阶段：将结果沿路径传回根节点"""
        while node:
            node.visits += 1
            node.wins += result
            node = node.parent
```

### MCTS 的关键优化

| 优化 | 问题 | 方案 |
|------|------|------|
| **渐进式扩展** | 一次性扩展所有子节点太慢 | 每次模拟只扩展一个子节点 |
| **RAVE** | 纯随机模拟质量低 | 记录动作在全局的胜率 |
| **AMAF** | 首次遇到的动作信息不足 | 借用其他分支的同动作数据 |
| **领域知识 playout** | 随机模拟太"笨" | 用轻量级启发式指导模拟 |
| **时间控制** | 迭代次数固定不灵活 | 基于时间预算动态调整 |

## 本技能对比

| 算法 | 需要评估函数 | 搜索策略 | 适合游戏类型 | 性能 |
|------|------------|---------|------------|------|
| **Alpha-Beta** | 是 | 深度优先 + 剪枝 | 棋类、确定性游戏 | ⭐⭐⭐⭐ |
| **Personality Search** | 是 | 个性偏好排序 | 移动策略、角色扮演 | ⭐⭐⭐ |
| **Interest Search** | 是 | 选择性深度 | 分支因子大的游戏 | ⭐⭐⭐⭐⭐ |
| **MCTS** | 否（用模拟替代） | 蒙特卡洛采样 | 围棋、卡牌、随机游戏 | ⭐⭐⭐ |

## 决策矩阵

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 棋类（象棋、国际象棋） | Alpha-Beta + 迭代加深 | 确定性、低分支因子 |
| 围棋 | MCTS | 分支因子极大，无法写评估函数 |
| 卡牌游戏（有随机性） | MCTS | 天然处理随机性 |
| 移动策略游戏 | Personality Search | 需要个性和表现力 |
| 回合制RPG | Interest Search | 分支多但大部分不重要 |
| 实时策略 | 浅层Interest Search | 时间预算紧张 |
