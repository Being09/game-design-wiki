# 第4章 约束：创意与完成的催化剂

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

**预先给自己施加约束**，再思考如何在约束内把游戏做有趣——这是催生区别于以往创意的有效技巧，也能提升点子独特性。约束还能加速完成（标准化流程减少开发负担）。Game Jam 是约束实践的典型场景。

## 约束类型（《Different Game Jam Formats》分类）

| 类型 | 说明 | 对创意/完成的作用 |
|---|---|---|
| **Tools** | 指定可用引擎 | 促完成（不花时间选工具） |
| **Assets** | 预先提供图文素材 | 促完成 |
| **Visuals** | 限制分辨率/颜色数 | **促创意**（极低像素如 16x16、1D 场地、甚至 1x1 两色） |
| **Goals** | 定义目标（"收集更多"等） | **促创意**（非常规目标如"毁灭自身""牺牲""利用敌人获胜"） |
| **Interaction** | 限制交互（"仅两按钮""仅鼠标""全键盘""双摇杆"） | **促创意**（核心，详见单按钮） |
| **Genre** | 指定类型（平台、roguelike） | 促完成 |
| **Time Limit** | 限时完成 | 促完成 |

> 促创意的约束聚焦于 **interaction / visuals / goals**；促完成的约束聚焦于 tools / assets / genre / time。

## 单按钮游戏（核心案例）

**定义**：所有玩法交互仅用一个按钮（非"摇杆+一按钮"）。

**优势**：操控直观，触屏易用——点击/长按屏幕任意处即可，绕开虚拟摇杆失灵问题；几乎无需说明。

**劣势**：引入动作变化难——连横向移动都需"按一次反转方向"这类非常规操控。

### 单按钮机制分类（作者自制游戏归纳）

按钮按下时执行的动作，分为"常规"与"独特"两类：

**常规动作**（缺乏单按钮特色）：反转移动方向/转90°、跳跃/拍打、射击。

**独特动作**（强烈印象单按钮特色）：
- **瞬移**（Teleportation）— 按键瞬移到最近的预设点（CYWALL）。
- **分裂**（Splitting）— 按键分裂导弹，持续按增攻击力但需引入不可击中目标防乱按（DIVARR）。
- **选择**（Choice）— 在特定点选择转向/跳台，需清晰呈现选择时机（JUMP ON）。
- **属性反转**（Attribute Inversion）— 类似磁极，按键切换属性，关键是选哪些属性及与环境如何交互（NS CLIMB）。
- **其他特殊**— 及时投放移动平台/梯子（LADDER DROP，单按钮俄罗斯方块式）；累加流动数字（NUMBER LINE，极特殊）。

详细机制综合见 [[单按钮游戏设计]]。

## 尺寸约束（代码 golf）

**js13kGames**：13KB 内做浏览器游戏。短代码创作的精髓是**信息密集之美**——用最少组件达成最大效果。

**作者的 jsgame256 实践**（256 字符内做游戏）：

游戏要素定义：计分系统、game over 条件、难度递增、（最好有）音效。

规则设定：
- 每帧 JavaScript update 函数，最长 256 字符。
- 引入 p5.js / Tone.js / lodash.range 便捷库。
- 给常用函数/变量取单字符别名：`M`=mouseIsPressed, `X/Y`=mouseX/Y, `R`=random, `A`=range, `N`=Tone.synth.triggerAttackRelease。
- 变量 `S`=分数（顶部显示），`T`=开局以来帧数（调难度）。

示例 springcar（点击跳跃的车）核心代码仅约 9 行紧凑逻辑。

**经验**：256 字符约 1 小时能做出基础游戏；但实现复杂动作很快超字符数，做出来 270 字符被迫砍机制很挫败——但"砍掉多余"也是挑战的一部分。**尺寸约束主要推荐给觉得"克服尺寸限制"本身有趣的人**，对多数人它更像枷锁而非助力。

## 核心论点

约束是一把双刃剑：适度的、契合个人 sensibility 的约束能激发创意并加速完成；过严的约束则难以为继。若想多做游戏，**建立自己的创作规则集、标准化"做什么、怎么做"**是稳健策略。详见 [[约束激发创意]]。
