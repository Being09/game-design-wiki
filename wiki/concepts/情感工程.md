# 情感工程

> 综合多本书的跨书主题文章（主要来源：《体验引擎》）。

## 定义

**情感工程**（Tynan Sylvester《体验引擎》的核心主题）是把游戏如何唤起玩家情感这件事，当作可分解、可设计、可优化的工程问题。其主干模型是一条因果链：**机制 → 事件 → 情感触发器 → 人类价值改变 → 情感 → 体验**。情感不是被设计师"传递"给玩家的，而是通过改变玩家所珍视的价值（财富、地位、关系、安全）被**诱导**出来的。

## 多本书视角对比

- **《体验引擎》**（[参考](../references/体验引擎.md)）：系统提供情感工程的工具集——基本情感触发器（学习、角色弧线、挑战、社交、财富、音乐等）、虚构层与机制层的分工、技巧范围与弹性挑战、决策的信息平衡、强化程序与动机叠加、博弈论与心理战。
- **《游戏设计艺术》**（[参考](../references/游戏设计艺术.md)）：Schell 用透镜方法触及同一主题（情感透镜、乐趣分类），但更偏"多视角反思"；Sylvester 更偏"因果链工程"。二者互补。

## 方法/模型

- **情感触发器**：能改变人类价值、从而唤起情绪的游戏元素（挑战、学习、社交、财富、音乐、场景特效、美等）。
- **虚构层（fiction layer）**：与机制并列运行的叙事/美术层，可与机制协同或对立，共同塑造氛围。
- **沉浸 = 心流 + 唤醒 + 虚构重标识**；驳斥"逼真即沉浸"的浸入谬论。
- **技巧范围与弹性挑战**：深度（上限）× 无障碍（下限），用弹性挑战随玩家水平伸缩，避免失败陷阱。
- **强化程序与动机叠加**：动机由多巴胺驱动（渴求而非享乐）；可变比例程序动机最强；叠加多个错峰程序消除"尘封时刻"，是长线留存的机制根源。
- **决策与信息平衡**：在信息匮乏（瞎猜）与信息过剩（无悬念）之间找到让选择有意义的区间。

## 应用

- 给每个核心机制标注"它改变玩家的哪项人类价值、唤起什么情感"。
- 用强化程序设计奖励节奏，避免单程序带来的弃游低谷。
- 多人游戏用博弈论与机制约束（而非道德说教）对冲破坏性玩家行为。

## 具体例子

**情感工程链的逐环追踪**（以"开宝箱"为例）：

| 链环 | 开宝箱机制的对应 |
|---|---|
| 机制 | 点击宝箱按钮 |
| 事件 | 宝箱打开，掉落道具 |
| 情感触发器 | 不确定性 + 稀缺奖励（可变比例程序） |
| 人类价值改变 | 财富（获得稀有道具）/ 地位（炫耀装备） |
| 情感 | 期待→惊喜（出货）/ 失望（垃圾）→ 渴求下一发 |
| 体验 | 上瘾循环 |

设计师不"传递"惊喜，而是通过"改变玩家对稀有道具的占有（价值）"诱导出惊喜。理解这链条就能精确调控——想减弱上瘾感，就降低"稀缺触发器"或加入"确定性保底"。

**强化程序的实战调参**：
- 固定比例（每 10 抽保底）：玩家知道"快到了"，临近时高度活跃，但拿完瞬间动机骤降（低谷）。
- 可变比例（随机出货）：无法预测，动机永不间断，最强但最易上瘾。
- 叠加程序：日常（固定间隔）+ 副本掉落（可变比例）+ 世界 Boss（可变间隔）+ 成就（固定比例）错峰，让任意时刻都有某个程序"快出奖"，消除低谷。

**弹性挑战 vs 固定难度**：固定难度只服务一个水平段——太简单高手无聊，太难新手受挫。弹性挑战让游戏内机制随玩家水平伸缩（如马里奥的辅助方块只对反复失败者出现、赛车游戏的追赶 rubberbanding）。深度（技巧上限）保证高手有挑战，无障碍（下限）保证新手能进入，弹性挑战在两者间动态匹配。

**浸入谬论的反例**：早期 VR 追求"画面越逼真越沉浸"，但 Sylvester 论证沉浸 = 心流 + 唤醒 + 虚构重标识。文字冒险游戏（《Zork》）靠想象达到强沉浸，而某些超写实游戏因机制割裂反而不沉浸。逼真是充分条件之一但远非充分——机制与虚构的协同才是关键。

**信息平衡的区间**（让选择有意义）：
- 信息太少（瞎猜）：选择无意义，纯靠运气，玩家挫败。
- 信息太多（全透明）：无悬念，最优解显然，选择机械化。
- 平衡区间：给足判断依据但保留不确定性——如牌类游戏看到部分牌、敌人显示血条但不显示策略。让玩家"基于不完全信息做有意义推断"。
