# 战术推理（Tactical Reasoning）


## 概述

战术推理是游戏AI从"能做"到"做好"的关键跃升。行为树和寻路让AI知道"怎么走"，战术推理让AI知道"去哪里、怎么打"。

从三个维度构建了战术AI的能力：

| 能力 | 核心问题 |
|------|---------|
| 哪个位置对我最有利？ |
| 哪条路径最安全？ |
| 什么时候用什么方式攻击？ |

### 适用场景

- 射击游戏中AI寻找掩体
- 策略游戏中单位的战场定位
- 潜行游戏中AI选择巡逻路线
- MOBA中AI的走位决策

## 核心概念

### 1. 战术位置选择

介绍了一个通用的战术位置评分框架：

```
PositionScore = Σ( Influence_i × Weight_i )
```

每个位置的评分由多个"影响因素"加权计算：

| 因素 | 说明 | 权重示例 |
|------|------|---------|
| **掩体质量** | 是否有全/半/无掩体 | 0.3 |
| **视野范围** | 能观察到多少关键区域 | 0.2 |
| **高度优势** | 是否处于高地 | 0.15 |
| **撤退路线** | 到安全区的距离 | 0.15 |
| **队友距离** | 是否在支援范围内 | 0.1 |
| **目标距离** | 是否在武器有效射程内 | 0.1 |

**位置查询语言（Position Query Language）：**

提出了一种声明式的查询语言来描述理想的战术位置：

```
// 示例：寻找压制位置
FIND Position WHERE
    cover_type >= HALF_COVER
    AND sightline_to("enemy_spawn") == TRUE
    AND distance_to("team") < 30
    AND distance_to("retreat_point") < 20
ORDER BY cover_quality DESC, sightline_count DESC
```

### 2. 战术寻路

战术寻路不是在"地图"上找最短路径，而是在"威胁图"上找最安全路径：

```
传统A*:     S →→→→→→→→→→→→→ G  ← 距离最短（但经过开阔地）
战术A*:     S →→ ↖ ↗ →→→→→ G  ← 路径更长但全程有掩体
```

**实现方案：**

```python
# 在 NavMesh 上叠加战术开销
def tactical_cost(navmesh_polygon, threat_map):
    base_cost = navmesh_polygon.area  # 基础移动开销
    threat = threat_map.get_threat(navmesh_polygon.center)
    exposure = navmesh_polygon.exposure_ratio  # 0=全掩体, 1=全暴露

    # 战术开销 = 基础 + 暴露风险
    return base_cost * (1.0 + threat * exposure * 2.0)
```

**Path 与 Safety 的权衡：**

| 策略 | 路径长度 | 安全性 | 适合 |
|------|---------|--------|------|
| 最短路径 | 最短 | 最低 | 安全环境 |
| 战术路径 | 中等 | 高 | 战斗环境 |
| 隐蔽路径 | 长 | 最高 | 潜行环境 |

### 3. 超越功夫圈

标题"Beyond the Kung-Fu Circle"指代一种常见的坏设计：NPC只能在固定圆圈范围内攻击。好的攻击管理系统应该：

**灵活的规则系统：**

```
AttackerAI
├── 攻击时机 (When?)
│   ├── 目标在武器射程内？
│   ├── 有清晰的射击线？
│   └── 自身状态允许？（非换弹、非硬直）
│
├── 攻击方式 (How?)
│   ├── 主武器（高伤害，慢速）
│   ├── 副武器（低伤害，快速）
│   ├── 近战（近身）
│   └── 特殊技能（有冷却）
│
└── 攻击目标 (Who?)
    ├── 最近的目标
    ├── 最弱的目标
    ├── 威胁最大的目标
    └── 玩家指示的目标
```

**攻击时机管理的关键维度：**

| 维度 | 描述 | 典型值 |
|------|------|--------|
| 攻击间隔 | 两次攻击之间的最小时间 | 0.5-3.0秒 |
| 攻击窗口 | 目标在有效范围内的持续时间 | 0.5秒以上才攻击 |
| 冷却时间 | 使用同一种攻击方式的间隔 | 5-10秒 |
| 切换延迟 | 切换目标/武器时的延迟 | 0.2-1.0秒 |

## 伪代码实现

```python
class TacticalPosition:
    """战场上的一个战术位置"""
    def __init__(self, x, y, cover_type, height_advantage):
        self.x = x
        self.y = y
        self.cover_type = cover_type  # NONE, HALF, FULL
        self.height_advantage = height_advantage
        self.sightlines = []          # 能看到哪些关键区域
        self.distance_to_retreat = float('inf')

    def evaluate(self, context) -> float:
        """评分：0.0 ~ 1.0"""
        score = 0.0
        score += self._cover_score() * 0.30
        score += self._sightline_score(context) * 0.25
        score += self._height_score() * 0.15
        score += self._retreat_score(context) * 0.15
        score += self._team_score(context) * 0.15
        return score

    def _cover_score(self):
        return {"NONE": 0.0, "HALF": 0.6, "FULL": 1.0}[self.cover_type]

    def _sightline_score(self, context):
        if not self.sightlines:
            return 0.0
        visible = sum(1 for sl in self.sightlines
                      if context.is_key_area_visible(sl))
        return min(1.0, visible / 3.0)

    def _height_score(self):
        return min(1.0, self.height_advantage / 5.0)

    def _retreat_score(self, context):
        return max(0.0, 1.0 - self.distance_to_retreat / 50.0)

    def _team_score(self, context):
        dist_to_team = context.distance_to_nearest_ally(self.x, self.y)
        return 1.0 if dist_to_team < 30 else max(0.0, 1.0 - (dist_to_team - 30) / 20.0)


class TacticalQuerySystem:
    """战术位置查询系统"""
    def __init__(self, navmesh, threat_map):
        self.navmesh = navmesh
        self.threat_map = threat_map

    def find_best_position(self, context, top_n=3):
        """找到最好的 N 个战术位置"""
        candidates = self._get_candidates(context)
        scored = [(pos.evaluate(context), pos) for pos in candidates]
        scored.sort(key=lambda x: x[0], reverse=True)
        return scored[:top_n]

    def find_tactical_path(self, start, end, context):
        """找到最安全的战术路径（而非最短路径）"""
        return self.navmesh.search(
            start, end,
            cost_fn=lambda poly: self._tactical_edge_cost(poly, context)
        )

    def _tactical_edge_cost(self, polygon, context):
        base = polygon.area
        threat = self.threat_map.get_threat(polygon.center)
        exposure = polygon.exposure_ratio
        return base * (1.0 + threat * exposure * 3.0)
```

## 实战要点

### 1. 战术评分更新频率

| 场景 | 更新间隔 | 原因 |
|------|---------|------|
| 阵地战 | 1.0-3.0秒 | 位置变化慢 |
| 运动战 | 0.3-1.0秒 | 快速变化需要及时反应 |
| 防守 | 2.0-5.0秒 | 位置相对固定 |

### 2. 战术信息的降级（Fog of War）

AI 不应知道所有信息。战术评分应考虑"AI 实际知道什么"：
- 视野内：使用精确评分
- 近期看到过：使用衰减评分
- 从未看到：使用默认评分

### 3. 个性参数

通过调整评分权重，可以创造不同"性格"的AI：
- **激进型**：高 sightline_score × 攻击范围权重
- **保守型**：高 cover_score × retreat_score
- **支援型**：高 team_score × 视野权重

## 反模式（NEVER 列表）

| # | 反模式 | 原因 |
|---|--------|------|
| 1 | ❌ 战术位置评分死板 | 不同武器/角色对"好位置"的定义不同 |
| 2 | ❌ 战术和移动耦合在一起 | 战术选"去哪"，移动管"怎么去" |
| 3 | ❌ AI 知道所有信息 | 战术决策应基于 AI 的实际感知，而非全知视角 |
| 4 | ❌ 忽略撤退路线 | 选择了进攻位置却没考虑怎么撤回来 |
| 5 | ❌ 每秒都重评战术 | 过于频繁的重评导致 AI 行为抖动 |
