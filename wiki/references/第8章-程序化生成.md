# 第8章 通过自动化提升开发

> 单章快速参考。综合自 ABA Games《Joys of Small Game Development》（CC BY 4.0）。原文见 [项目仓库](https://github.com/abagames/joys-of-small-game-development-en) / [在线阅读](https://abagames.github.io/joys-of-small-game-development-en/)。

## 概述

**程序化生成（Procedural Generation）**——让程序生成像素画、音效、BGM、关卡——是"少费力做游戏"的强力工具。质量虽不及手工，但反复生成直至满意，能以最小成本获得必要资源；随机关卡算法还让每次游玩体验独特，提升重玩价值。

## 像素画自动生成

| 方法 | 原理 | 优劣 |
|---|---|---|
| **部件组合 + 变色** | 组合大量预设部件并改色生成新像素画（如 pixeldudesmaker） | 只能造特定角色变体；无匹配生成器就得不到想要的角色 |
| **随机点变异** | 在基础像素画上随机开关点，制造各种图案（Pixel Spaceships） | 更灵活，换基础图就能造各类角色；作者 pixel-art-gen 用此 |
| **生成式 AI** | 用 prompt 生成（DALL-E 3 / Bing Image Creator） | 擅长大图，小图（16x16）不适合；prompt "32x32 网格 16 色"也未被精确遵守 |

**随机点变异法的适用边界**：随机性高，擅长**自由形态角色**（如飞船）；但对有约束的角色（侧视怪物，需明确的眼睛/手臂/腿）表现差。**无万能方案，各有长短**。

> 未来生成式 AI 或许能精确生成小像素画及其动画。

## BGM 与音效自动生成

### BGM：何谓"适合小游戏的 BGM"
复古街机（Xevious、Dig Dug，1980 年代初）的 BGM：**4-8 小节的短乐句反复**。

### 工具对比

| 工具 | 类型 | 适用性 |
|---|---|---|
| **WolframTones** | 算法生成 | 便利但难复现老游戏的经典乐句（如纯五度低音线、十六分音符） |
| **Jukebox / MusicLM**（OpenAI/Google） | 生成波形而非乐谱 | 不适合复古游戏；MusicLM 有 8-bit 样本偏 chiptune 但表达过丰 |
| **Magenta / Magenta.js**（Google） | ML 应用于音乐 | **最佳**——给乐谱用 MusicRNN 自动续写乐句 |

### 作者实践：short-vgm-generator
用 Magenta.js：给左侧一段复古游戏短谱 → 点 Generate → 右侧生成相似 vibe 的新乐句（仅旋律+贝斯轨）。
- **多轨分别生成**，再调整（确保贝斯与旋律频率比约 1:4，避免不和谐）。
- 取消勾选可关闭调整，生成更自由乐句。

### 音效
MusicRNN 面向旋律，不适合音效。音效多用**基于波形的方案**，很多源自 **sfxr**——给动作游戏加简单音效。

## 益智关卡自动生成

关卡质量直接影响玩法，**非动作益智游戏（如推箱子 Sokoban）尤其需要"解起来有趣"的关卡**。

### 推箱子生成经典三步法
1. 造一个不放箱子的空房间。
2. 把箱子放到正确位置。
3. **逆向模拟**游戏把箱子散开（第 3 步保证玩家"拉"箱子能移回正确位）。

进阶：玩家卡住时回溯搜其他摆法；用指标（直线推箱次数、换推另一箱的次数）让箱子尽量远离正确位以增加任务量。

**PuzzleScript 通用法**：把 PuzzleScript 规则分析器的游戏对象特性传给关卡生成器 + **评估器**——评估器判定关卡可解性/趣味性（不同于逆向模拟法）。

### 作者实践

**slickslack**（推箱子+Pengo 混合，弹蓝色方块到黄色正确位）：
逆向模拟法的变体——先造**全墙房间**，逆向弹箱子的同时**挖出墙体**：
1. 全墙（或可墙可空）房间。
2. 箱子放正确位。
3. 任意方向弹任意距离。
4. 在弹出方向起点放墙确保箱子停住。
5. 确认弹出区域为空（挖墙）。

**调优**（因无玩家角色、箱子可自由弹，需增加谜题感）：
- 弹时优先选能被现有墙/其他箱子挡住的方向，少造新墙。
- 弹后尽量让箱子与其他箱子相邻。

**HAKOSAN**（方向键同时滑三个角色到红色终点）：同样从终点逆向滑动生成。

### 核心方法
**"从答案逆向模拟"** 适用面广。加上 PuzzleScript 论文式的**评估器**（箱子与正确位距离、玩家移动数、应用规则数等指标）筛选趣味性，能进一步提升。指标虽未必准确反映"解谜乐趣"，但能滤掉明显无趣的关卡。
