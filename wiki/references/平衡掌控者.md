# 平衡掌控者 — 参考

> 单书快速参考。综合自《平衡掌控者——游戏数值战斗设计》（似水无痕，电子工业出版社）。原著信息见 [豆瓣读书](https://book.douban.com/subject/35488031/)。

## 概述

《平衡掌控者——游戏数值战斗设计》（似水无痕，电子工业出版社 2017）是一线数值策划实战工具书。作者以真实参与国内游戏研发的经验为底，案例均来自真实游戏而非数学应用题，明确将数值策划工作拆分为「**设计层**」（构建战斗/经济数学模型、把握平衡与节奏）与「**实现层**」（精通 Excel、维护数据表格、必要时用 VBA 模拟）。本书聚焦 RPG 战斗系统，围绕角色属性、战斗公式、技能、装备、随机掉落、数据结构与 VBA 战斗模拟，给出从理论模型到 Excel 落表的完整链路。

## 章节骨架

### 第1章 数值策划的定位
先介绍研发团队四大工种及策划内部细分（系统、关卡、文案、资源、运营、UI、脚本、数值）。随后归纳数值策划职责：制定规则公式、建立数学模型、搭建经济与战斗系统、填写维护数据表格、配合 QA 调试。作者把招聘需求提炼为「设计层」与「实现层」两个方向，并分别给出提升建议。

### 第2章 数值策划的基础知识
讲解设计层基础：游戏类型分类（RPG/ACT/AVG/SIM/策略等及子类）、基于心理需求的玩家四分类（杀手型、成就型、探索型、社交型，指出中国市场以杀手型与成就型为主）。以《龙与地下城》骰子判定体系（D20、属性调整值表、命中-暴击-防御结算）追溯 RPG 数值判定原型。提出数值策划应具备的五项素质：数学功底、逻辑思维、数字嗅觉、吃苦耐劳、经验积累。

### 第3章 数值策划相关 Excel 知识
实现层核心工具章。从学习方法、文件格式（推荐 .xlsb）、相对/绝对引用讲起，系统讲解运算符优先级，并逐一解析三十余个常用函数（ABS、AVERAGE 系列、CEILING/FLOOR、COUNT 系列、IF、INDEX、INDIRECT、LOOKUP/MATCH、OFFSET、ROUND、SUM 系列等）。此外涵盖名称管理器与数组公式。强调这是数值工作 80% 时间所依赖的基本功。

### 第4章 设计层进阶之路
全书方法论核心。先以「标准人 + 职业定位系数」设计基础属性，引入 DPS、有效生命（EHP）、角色强度概念。战斗公式部分系统对比：战斗流程（前摇/后摇/CD/公共 CD/攻速）、逐步判断 vs 圆桌理论、闪避与暴击的减法/除法公式、伤害的减法 vs 乘除法（护甲减免率、攻/(攻+防)）公式及三条曲线对比。随后讲解技能分类与技能输出序列的价值衡量、控制/BUFF 技能价值、装备主属性与攻击装/防御装划分，以及随机掉落方式（计数随机、逐个百分比、权重掉落组、进阶掉落、木桶原理掉落）与战斗中多次取均值平滑随机。

### 第5章 实现层进阶之路
先讲数据结构：静态数据与动态数据、前后端数据职责（后端运算/产出，前端显示）、配置文件格式（CSV/XML/JSON）与多版本维护。静态数据简析给出角色基础属性表、装备属性表、技能表、BUFF 表、怪物表的字段规范。实战设计以一款 4 职业（战法牧刺）60 级上限的简易 MMORPG 为例，按角色属性→战斗公式→公式曲线→装备设计→技能设计（DPS 增量法）→药品恢复→怪物属性（由角色属性反向推导）的顺序完整走通数值落表。

### 第6章 VBA 知识及实战模拟
前半部分讲解 VBA 基础：宏、编辑器、模块与过程、常量变量数据类型、运算符、数组、参数传递、对象/属性/类。后半部分用 VBA 将第 5 章设计的战斗实现为可运行模拟器：构建战斗角色数据、技能、BUFF、角色技能、战斗日志等数据表，以 0.1 秒为心跳时间循环，实现技能冷却计算、命中（phit）/暴击（pcri）/行为类型/伤害计算等函数，并对不同技能系数（60% vs 120%）做模拟对比。作者指出模拟可提升战斗预期准确性但无法涵盖玩家操控等全部因素，最终仍需玩家测试反馈反复调优。

## 核心方法论

**标准人与职业定位法**：先定一个「标准裸体人」的生命值与攻击（由预期基础战斗时间反推比值），再用二维职业定位表对各职业各项能力打分、以系数控制职业差异，使所有属性都基于标准人按比例推导。这是后续属性、装备、怪物设计的统一基准。

**战斗公式体系（逐步判断 vs 圆桌理论 + 三种伤害公式）**：用逐步判断（逐级条件衰减）或圆桌理论（高优先级占桌）决定攻击结果类型；伤害计算在减法公式（数值敏感、反馈强）、乘除法 1（护甲换算固定减免率，代表《魔兽世界》）、乘除法 2（攻击×攻击/(攻击+防御)，代表新生代国产游戏）之间按项目目标取舍。

**属性价值统一汇率法**：把攻防属性折算为通用货币——攻击类看提升多少 DPS，防御类看提升多少 EHP，按预期战斗时长平衡输出与生存；属性随等级变化，汇率也随等级变动，需固定在特定条件（如标准人）下才能确定属性价值。

**技能 DPS 增量设计法**：把普通攻击视为持续输出基线，技能视为对 DPS 的增量；用「技能攻击 = (增量DPS - 普攻DPS) × 时间」反算技能瞬时攻击。输出型技能系数取 60%~120%，AOE 约 20%，控制/冲锋等功能性技能折损伤害并设长 CD。

**Excel 数据落表 + VBA 战斗模拟闭环**：数据按静态/动态、前后端职责分类，策划用 Excel 维护并转为配置；角色、装备、技能、BUFF、怪物各成独立表格。最后用 VBA 以 0.1 秒心跳循环驱动，将命中/暴击/伤害公式写成函数，输出文字战报，通过调整参数反复模拟以逼近预期战斗节奏。

## 关键术语

- **设计层 / 实现层**：数值策划工作的两个方向，前者构建数学模型并把握平衡节奏，后者精通工具、维护数据、调试模型。
- **一级属性 / 二级属性**：一级（力量、智力、敏捷等）可成长加点，二级（生命、物攻、命中、闪避、暴击等）依附一级或装备加成，通过转换系数表关联。
- **标准人**：设定的基准属性模型，各职业属性以标准人乘以系数得出，是衡量属性价值与怪物强度的参照。
- **DPS**：每秒输出伤害，衡量职业/技能的输出能力。
- **有效生命（EHP）**：综合防御、闪避、暴击等所有战斗因素后防御方可承受的等效生命数值。
- **逐步判断**：按命中→暴击→普通攻击逐级条件判定攻击结果，高优先级逐级衰减低优先级的实际概率。
- **圆桌理论**：把各攻击结果按优先级占据固定「圆桌」（概率和为 100%），挤出底层结果前各高优先级结果彼此不影响；源自《魔兽世界》。
- **公共 CD（GCD）**：对所有技能施加的固有冷却，限制单位时间指令量与输出量。
- **减法公式**：伤害 = 攻击 - 防御，数值敏感、反馈直接，不破防时需做保底处理。
- **乘除法公式 1**：伤害 = 攻击×(1-护甲减免率)，减免率 = 护甲/(护甲+等级×K1+K2)，代表《魔兽世界》。
- **乘除法公式 2**：伤害 = 攻击×攻击/(攻击+防御)，强调攻击价值，代表新生代国产 MMORPG。
- **权重掉落组式掉落**：母集与子集双层结构，按权重随机选择道具，可精确配置掉落组合。
- **进阶掉落**：在权重掉落基础上增加前置计数判断（最小/最大进阶次数），保证某物品在区间内必然掉落。
- **木桶原理掉落**：根据道具空缺量与空缺率赋予额外权重，使玩家最短缺的道具掉落概率更高。
- **静态数据 / 动态数据**：前者稳定不变（各级基础生命值），后者随事件变化（当前生命值）。
- **技能输出序列**：玩家按 CD 顺序循环释放技能形成的输出节奏，衡量整体输出能力的基础单位。
